How it works

From signup to decision in under 2 minutes.

ReguFlow sits between your product and your data providers. Every applicant flows through a consistent, transparent decisioning pipeline.

  • 1
    Collect data from your existing forms. Use our API or SDKs to stream application data, documents, and device context into ReguFlow as the user completes your onboarding.
  • 2
    Run KYC & AML checks in parallel. Kick off ID verification, sanctions & PEP screening, fraud rules, and custom logic based on customer type or geography.
  • 3
    Score risk & decide automatically. Combine scores and rules into a final decision: approve, reject, or escalate to a human reviewer queue with all the context they need.
  • 4
    Stream decisions back to your product. We return a final decision and explanation so your UI can instantly move the user forward, ask for more info, or deny access.
Example: decision API
// 1. Send applicant data to ReguFlow
POST /v1/kyc/decision
body: {
  customer_id: "usr_123",
  name: "John Doe",
  dob: "1993-05-14",
  country: "DE",
  document: {
    type: "passport",
    number: "C1234567"
  }
}

// 2. ReguFlow runs KYC, AML & risk rules

response = {
  decision: "approve",
  risk_score: 0.21,
  reason: "No watchlist hits, low geo-risk.",
  checks: ["document_verification", "pep_screening"]
}
